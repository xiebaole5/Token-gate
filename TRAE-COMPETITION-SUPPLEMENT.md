# TokenGate 大赛补充资料

## 帖子标题

学习工作｜TokenGate：给 AI 工具装一个“花钱前先问人”的预算闸

## 500 字开头版

AI 编程越来越像开着一辆没有仪表盘的车：模型在调用、token 在消耗、多个工具同时运行，但用户往往不知道钱花到哪里去了，也不知道什么时候该停。尤其是 vibe coding 用户，一边用 TRAE、Claude、Cursor、Codex CLI 等工具快速试错，一边又要去各个平台查余额、估成本、看账单，最后很容易变成“项目还没做完，额度先烧穿”。

TokenGate 想解决的就是这个真实问题：它不是一个普通记账面板，而是一个本地优先的 AI token 开销管理工具。用户可以录入不同模型的单价、项目归属和预算上限；当一次调用即将接近或超过预算时，TokenGate 会弹出“预算闸”，让用户明确选择放行、拦截或记录原因。它的核心理念是：AI 可以执行，但花钱这件事必须先经过人的边界。

本 Demo 已经跑通从 AI 工具扫描、Provider 创建、代理监听、usage 入库、额度扣减到端到端测试报告的完整链路。开发过程全程使用 TRAE 国际版与中国版：国际版参与早期需求拆解，把“按量付费 API 额度痛点”升级成“额度账户”概念；中国版负责核心组件实现、联调测试和结果验证。两版都属于 TRAE 体系，符合大赛“全程使用 TRAE Work 或 TRAE IDE”的要求。

我做 TokenGate 的原因很简单：AI 时代不缺会帮你花 token 的工具，缺的是一个帮你守住预算、守住决策权、守住本地隐私的工具。

## 30 秒讲解词

TokenGate 是一个给 AI 创作者用的本地 token 预算闸。它会扫描本机 AI 工具，建立 Provider 和模型单价，实时记录调用消耗，并把余额、项目成本和模型排行展示在驾驶舱里。最核心的是“预算闸”：当一次调用接近或超过预算时，它不会默认放行，而是让用户确认是否继续。它解决的不是记账，而是 AI 时代的成本失控和决策权外包问题。

## 评委看点

- **真实痛点**：按量付费 API、多个 AI 工具、余额分散、账单难追踪。
- **真实闭环**：扫描工具、建 Provider、代理监听、usage 入库、额度扣减、测试报告已跑通。
- **AI 必要性**：TRAE 不只是写代码，还参与需求升级、组件生成、端到端验证。
- **本地优先**：预算、usage、模型价格和项目记录优先留在本机，降低隐私外泄。
- **产品差异**：从“事后看账单”变成“花钱前先问人”。
